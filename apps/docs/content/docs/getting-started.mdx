## Install

```package-install
@wingleeio/mugen react react-dom
```

React 18.2+ or 19 is required.

mugen measures text with [`@chenglou/pretext`](https://www.npmjs.com/package/@chenglou/pretext)
and depends on it directly, so it's installed for you ‚Äî you don't add it
yourself. That dependency is **pinned to an exact version** (`@chenglou/pretext`
is pre-1.0 and its API still moves between releases), so every mugen install
resolves the same, known-good pretext; a new pretext is adopted only by a new
mugen release. If you reach for pretext's escape hatches, import them from
`@wingleeio/mugen` (e.g. `prepareText`, `measureText`) rather than installing
pretext separately, so you stay on that one pinned copy.

## Your first list

A list is two pieces:

- **`useMugenVirtualizer({ items })`** ‚Äî a hook that creates and owns the list
  *instance* (its measurement engine and scroll position).
- **`<MugenVList>`** ‚Äî the component. You give it the `instance`, a `getKey`, and
  a `render` that returns each row as a tree of **primitives**.

```tsx twoslash
import { MugenVList, Text, VStack, useMugenVirtualizer } from '@wingleeio/mugen';

interface Comment {
  id: string;
  author: string;
  body: string;
}

export function Comments({ comments }: { comments: Comment[] }) {
  const list = useMugenVirtualizer({ items: comments });
  return (
    <MugenVList
      instance={list}
      getKey={(c) => c.id}
      // List-level text defaults ‚Äî every <Text> inherits them (and can override).
      font="15px Inter"
      lineHeight={22}
      maxW="2xl"
      render={(c) => (
        <VStack gap={4} padding={12}>
          <Text font="600 15px Inter">{c.author}</Text>
          <Text>{c.body}</Text>
        </VStack>
      )}
    />
  );
}
```

That's a fully virtualized list. No `react-virtuoso`, no measure-on-mount, no

## The item type is inferred

`useMugenVirtualizer({ items })` infers the row type from `items`, and it flows
through `getKey` and `render`. Pass it explicitly if you need to ‚Äî
`useMugenVirtualizer<Comment>({ items })`.

## The font must be measurable

pretext measures against the browser's font engine, so `font` must be a **named
font** in canvas shorthand (`"16px Inter"`, `"600 15px Inter"`). `system-ui` is
rejected ‚Äî its metrics differ per OS, so heights would drift from what the user
sees.

```ts twoslash
import type { Font } from '@wingleeio/mugen';
// ---cut---
const ok: Font = '16px Inter'; // ‚úì measured against the font engine
const sys: Font = '16px system-ui'; // type-ok, but throws at runtime (per-OS metrics)
```

If the font loads asynchronously (most web fonts do), mugen re-measures
automatically once `document.fonts` settles ‚Äî no layout shift.
