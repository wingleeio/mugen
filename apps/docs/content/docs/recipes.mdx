## AI chat with thinking & steps

A modern assistant transcript is a stress test for a virtualizer: every turn is a
different height, reasoning traces collapse and expand, and the newest turn keeps
growing as it streams. It maps cleanly onto mugen — each turn is a pure
`item → tree` that uses all three hooks. `useMugenState` collapses the “Thought
for…” trace (toggling re-measures just that row); `useMugenEffect` lets the final
turn “think”, then stream its answer in; and because heights are computed, a turn
far off-screen already knows its exact size. Expand a reasoning trace and watch the
list re-flow without losing your place:

## Deep link into a row

Open a specific row's state and scroll to it — the scroll lands pixel-exact
because the off-screen height is already known. Drive expansion from the item data
so it works before the row ever mounts:

```tsx twoslash
import { MugenVList, Text, VStack, useMugenVirtualizer } from 'mugen';

interface Thread {
  id: string;
  title: string;
  body: string;
  open: boolean; // expansion lives in the data
}

export function Threads({ threads, focusId }: { threads: Thread[]; focusId?: string }) {
  const list = useMugenVirtualizer({ items: threads });

  const openAndReveal = (id: string) => {
    // flip `open` for this thread in your own state, then:
    list.scrollToItem(id, { behavior: 'smooth', align: 'center' });
  };
  if (focusId) openAndReveal(focusId);

  return (
    <MugenVList
      instance={list}
      getKey={(t) => t.id}
      font="15px Inter"
      lineHeight={22}
      render={(t) => (
        <VStack gap={4} padding={12}>
          <Text font="600 15px Inter">{t.title}</Text>
          {t.open ? <Text>{t.body}</Text> : null}
        </VStack>
      )}
    />
  );
}
```

See [Scrolling](/docs/scrolling) for the full `scrollToItem` API.

## Shrink-to-fit bubbles

Chat bubbles that hug their text instead of filling the column:

```tsx twoslash
import { Text, VStack } from 'mugen';

interface Msg {
  id: string;
  text: string;
}

export function Bubble(msg: Msg) {
  return (
    <VStack padding={10}>
      <Text shrink>{msg.text}</Text>
    </VStack>
  );
}
```

`shrink` measures the text at its natural (tightest) width via pretext's
`naturalWidth`, and renders with `width: fit-content` so the browser shrink-wraps
to the same width.
